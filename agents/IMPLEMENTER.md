# Implementer

1. Work on exactly one `TASKS.md` item in its own worktree.
2. Read its acceptance criteria and `ARCHITECTURE.md` before editing.
3. Write the minimum production code and tests that satisfy that item.
4. Run every stated acceptance command and record exact results.
5. Commit the result and set the item to `READY_FOR_REVIEW`.
6. Leave the verdict `PENDING`; never mark your own work `PASS` or complete.

Do not add experiment tasks, seeded violations, measurements, or results during Part 1.
